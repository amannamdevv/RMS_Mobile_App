1. Product Overview

The TPMS (Telecom Power Monitoring System) is an enterprise-grade IoT platform developed by Shroti Telecom for real-time remote monitoring and security of passive infrastructure. Derived from its core 2026 unified codebase, the system combines hardware-level sensing with a powerful Laravel REST API and a CodeIgniter-based dashboard to provide 24/7 visibility into site health, security, and power consumption.

2. Key Features (Verified from System Code)

  • Real-Time Site Status Track: Differentiates between 'Active' (Communicating) and 'Non-Comm' (Stopped) sites using last-received data timestamps.
  • Multi-Level Alarm Classification: Groups alarms into 7 Priority Levels: Critical, Major, Minor, Information, Reset, Clear, and Site Alarm.
  • Advanced Intrusion Detection: Monitors physical site security including Door Open, Cabinet Open, and Enclosure Tampering.
  • Automated Escalation Matrix: Stores L1 to L5 contact numbers per site for instant automated alerts (Mobile App, WhatsApp, Robo-Calls).
  • Energy Management: Tracks 5-channel Energy Meter data (KWH) and calculates Site Run Hours for EB, DG, and Battery.
  • Integrated NOC Dashboard: Enterprise-wide visualization with state-wise India Heatmap and Pie/Column charts for alarm status.
  • Role-Based Access: 6 tiers of user hierarchy (NationalLevel down to LevelOne) ensures data security and role accountability.
  • Field Evidence Support: Capability to upload and view site/battery installation images directly in the site details module.

6. System Administration & Escalation

TPMS manages a structured response system for every alarm event:

  • User Hierarchy: National Admin → Circle Head → District Manager → Cluster Lead → Site Technician.
  • Alert Channels: System triggers notifications via OneSignal (Push), WhatsApp communication, and Robo-Call voice alerts.
  • Acknowledgement System: Tracks 'Call ACK' status for every alarm to ensure field team responsiveness.
  • Site Master: Stores IMEI, SIM details, installation dates, and technician mapping for every tracker.
